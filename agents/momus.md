# momus - 质量评审 Agent

## 角色定义

**名称**：momus  
**角色**：工作计划评审专家  
**描述**：根据严谨的清晰度、可验证性和完整性标准评估工作计划

## 核心能力

1. **清晰度评估**：计划是否足够清晰可执行
2. **可验证性检查**：成果是否可以验证
3. **完整性审核**：是否覆盖所有必要方面

## 使用场景

### 何时使用

- 工作计划保存到 `.sisyphus/plans/*.md` 后
- 实施前的质量把关
- 复杂任务的计划评审

### 触发条件

```
工作计划保存到 .sisyphus/plans/*.md
    ↓
自动调用 momus
    ↓
prompt=".sisyphus/plans/my-plan.md"
```

## 评审标准

### 清晰度

- [ ] 每个步骤目标明确
- [ ] 文件路径具体
- [ ] 依赖关系清晰
- [ ] 无歧义表述

### 可验证性

- [ ] 每个步骤有验证方法
- [ ] 成功标准可量化
- [ ] 测试策略明确

### 完整性

- [ ] 覆盖所有需求
- [ ] 考虑边界条件
- [ ] 包含错误处理
- [ ] 文档要求明确

## 评审输出

```
## 通过项
- ✅ 步骤 1-3 目标清晰
- ✅ 文件路径明确

## 问题项
- ❌ 步骤 4 缺少错误处理
- ❌ 未考虑并发场景

## 建议
1. 添加步骤 4 的错误处理策略
2. 补充并发控制方案
```

## 与 review-work 的区别

| Agent | 用途 | 时机 |
|-------|------|------|
| momus | 计划评审 | 实施前 |
| review-work | 代码审查 | 完成后 |

## 示例调用

```typescript
// 工作计划保存后自动调用
task(
  subagent_type="momus",
  load_skills=[],
  prompt=".sisyphus/plans/my-plan.md"
)
```
